В Python некоторые типы данных называются **неизменяемыми** (или **иммутабельными**). Это значит, что после создания их содержимое нельзя изменить. К таким примитивным типам относятся `int`, `float`, `str` и `bool`.

Рассмотрим это на примере строк.

### Методы строк не меняют оригинал

Когда мы вызываем метод у строки, кажется, что мы меняем ее. Например, переводим в верхний регистр.

```python
text = 'hexlet'
text.upper()
print(text)  # => hexlet
```

На самом деле метод `upper()` **возвращает новую строку** в верхнем регистре, а оригинальная строка `text` остается прежней.

```text
text = 'Python'

text.upper()  →  'PYTHON'  (новая строка)
text          →  'Python'  (не изменилась)
```

Чтобы не потерять результат, сохраним его в переменную.

```python
text = 'hexlet'
new_text = text.upper()
print(new_text)  # => HEXLET
```

Если вы не сохраните результат метода в новую переменную, вы его просто потеряете.

Другие методы работают так же.

```python
text = '   hi   '
cleaned = text.strip()
print(cleaned)  # => 'hi', результат не содержит пробелов
print(text)     # => '   hi   ', строка не изменилась, она содержит пробелы
```

Метод `strip()` вернул новую строку без пробелов, но сам `text` не изменился.

Методы не изменяют оригинальную строку, потому что строки **не могут быть изменены после создания**. Это решение принято в Python специально. Случайно изменить значение строки невозможно. Это же упрощает **многопоточный** код и позволяет Python переиспользовать одинаковые строки в памяти, экономя ресурсы.

Позже вы увидите, что изменяемость и неизменяемость являются важными свойствами, особенно при передаче данных в функции и работе с коллекциями.

### Переприсвоение переменной

Результат метода можно записать обратно в ту же переменную.

```python
text = '   hexlet   '
text = text.strip()
print(text)  # => 'hexlet'
```

Это уместно, когда суть данных не меняется. После `strip()` это тот же текст, просто чище. Если результат метода представляет другую сущность, стоит дать ей отдельное имя.

```python
full_name = 'John Doe'
header_name = full_name.upper()
```

`full_name` — имя человека. `header_name` — его вариант для заголовка. Это разные вещи, и хранить их в одной переменной было бы путаницей. Название переменной должно отражать смысл данных, которые в ней лежат.
